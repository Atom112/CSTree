## 编译器——把人类语言翻译成机器语言

你写 `int sum = a + b * 2;`——CPU 不认识这行字。它只认 0 和 1。**编译器（Compiler）** 把这行代码变成 CPU 能执行的机器指令。

> 🏫 **类比：英译中翻译**
> 把一篇英文论文翻译成中文：
> 1. **词法分析**：认识每个英文单词（apple→苹果，eat→吃）
> 2. **语法分析**：理解句子结构（"I eat apple" → 主谓宾）
> 3. **语义分析**：检查逻辑（"I eat apple"正确，"I eat rock"不合理但语法对）
> 4. **代码生成**：写出对应的中文
> 5. **优化**：润色译文，让它更通顺

## 编译器的整体结构

```
源程序（.c 文件）
    │
    ▼
┌─────────────────┐
│  词法分析器      │ → Token 序列
│  (Lexer)        │
└────────┬────────┘
         ▼
┌─────────────────┐
│  语法分析器      │ → 抽象语法树（AST）
│  (Parser)       │
└────────┬────────┘
         ▼
┌─────────────────┐
│  语义分析器      │ → 带类型标注的 AST
│  (Semantic)     │
└────────┬────────┘
         ▼
┌─────────────────┐
│  中间代码生成    │ → IR（三地址码）
│  (IR Gen)       │
└────────┬────────┘
         ▼
┌─────────────────┐
│  优化器          │ → 优化后的 IR
│  (Optimizer)    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  目标代码生成    │ → 汇编/机器码
│  (Code Gen)     │
└─────────────────┘
```

### 前端 vs 后端

编译器分为**前端**和**后端**：

```
源语言 ─→ 前端 ─→ 中间表示（IR） ─→ 后端 ─→ 目标机器码
```

**前端**：与源语言相关（C 前端、Java 前端、Go 前端）→ 产生 IR
**后端**：与目标机器相关（x86 后端、ARM 后端、RISC-V 后端）→ 从 IR 生成机器码

> 💡 **LLVM** 的核心思想就是"通用 IR"——写一个新语言只需要实现前端（源语言 → LLVM IR），写一个新架构只需要实现后端（LLVM IR → 目标机器码）。C/C++/Rust/Swift 共享 LLVM 后端。

## 编译器 vs 解释器

| 对比 | 编译器（Compiler） | 解释器（Interpreter） |
|------|------------------|---------------------|
| **工作方式** | 一次翻译完，生成可执行文件 | 逐行翻译执行 |
| **执行速度** | 快（编译后直接运行机器码） | 慢（每次都要翻译） |
| **启动时间** | 慢（需要先编译） | 快（即时运行） |
| **典型代表** | C、C++、Rust、Go | Python、JavaScript、Ruby |

> 💡 Java 和 C# 是**混合模式**：先编译成字节码（不是机器码），运行时再用 JIT（Just-In-Time）编译成机器码——兼得编译速度和跨平台。

## 编译器的历史

| 年代 | 里程碑 |
|------|--------|
| 1950s | 第一个编译器——Grace Hopper 的 A-0 系统 |
| 1960s | FORTRAN 编译器（第一个真正高效的编译器） |
| 1970s | C 编译器、Unix 的 cc、Pascal 编译器 |
| 1980s | GCC（GNU C Compiler）诞生 |
| 1990s | Java 字节码 + JIT |
| 2000s | LLVM 架构、Clang 编译器 |
| 2010s | Rust 编译器（基于 LLVM） |

## 小结

| 阶段 | 输入→输出 | 核心任务 |
|------|----------|---------|
| **词法分析** | 字符流→Token 序列 | 识别单词 |
| **语法分析** | Token 序列→AST | 理解结构 |
| **语义分析** | AST→带类型的 AST | 检查逻辑 |
| **IR 生成** | AST→中间表示 | 平台无关代码 |
| **优化** | IR→更好的 IR | 跑得更快 |
| **代码生成** | IR→目标机器码 | 输出结果 |
