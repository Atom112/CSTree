## 五大原则

| 缩写 | 原则 | 含义 |
|:----:|:----:|------|
| **S** | 单一职责 | 一个类只做一件事 |
| **O** | 开闭原则 | 对扩展开放，对修改关闭 |
| **L** | 里氏替换 | 子类型必须能替换父类型 |
| **I** | 接口隔离 | 大接口拆小 |
| **D** | 依赖反转 | 依赖抽象，不依赖具体实现 |

## 小结

| 原则 | 一句话 |
|:----:|--------|
| **SRP** | 一个类一个职责 |
| **OCP** | 扩展不修改 |
| **LSP** | 子类能替换父类 |
| **ISP** | 接口不要太大 |
| **DIP** | 依赖接口不要依赖实现 |

**为什么先学这个？** 好的设计需要[[software-testing|软件测试]]来验证。
