## 敏捷宣言

```
个体和互动    高于 流程和工具
可工作的软件  高于 详尽的文档
客户合作      高于 合同谈判
响应变化      高于 遵循计划
```

## Scrum 角色

| 角色 | 职责 |
|:----:|------|
| **Product Owner** | 确定需求优先级 |
| **Scrum Master** | 确保流程顺利运行 |
| **开发团队** | 实际完成开发工作 |

## 小结

| 概念 | 要点 |
|:----:|------|
| **Sprint** | 2-4 周迭代周期 |
| **每日站会** | 15 分钟同步进度 |
| **回顾会议** | Sprint 结束后反思改进 |

**为什么先学这个？** 敏捷是开发流程。实践工具——[[version-control|版本控制（Git）]]。
