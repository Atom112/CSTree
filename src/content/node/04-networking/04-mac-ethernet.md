## 多台电脑共用一根线——怎么不吵架？

早期以太网使用**同轴电缆总线**——所有电脑都连在同一根线上。如果两台电脑同时发送数据——信号会冲突，数据全坏。

> 🏫 **类比：多人同时说话**
> 一群人在一个房间里聊天。如果两个人同时开口——谁都听不清。解决方案：
> 1. **先听**——别人在说话时先等（载波监听）
> 2. **再说**——没人说话了再开口（多点接入）
> 3. **边听边说**——说的同时检测是否和别人冲突（冲突检测）

## CSMA/CD 协议

**CSMA/CD（Carrier Sense Multiple Access with Collision Detection）**——载波监听多点接入/冲突检测：

```
计算机 A 准备发送
    │
    ▼
监听信道是否空闲 ←── 是 → 发送数据
    │                        │
    │ 忙                     │ 发送时继续监听
    ▼                        │
等待随机时间 ←── 检测到冲突 ──┘
                     │
                发送拥塞信号（通知所有人）
                     │
                等待随机退避时间
                     │
                     ▼
                  重试（最多 16 次，失败则丢弃）
```

**退避算法（指数退避）：**
- 第一次冲突：等待 0-51.2µs 之间的随机时间
- 第二次冲突：等待 0-102.4µs
- 第 n 次冲突：等待 0 到 (2^n × 51.2µs) 之间的随机时间

> 💡 现代以太网使用**全双工交换机**——每个设备独享一条信道，不需要 CSMA/CD。只有在用集线器（Hub）的半双工模式下才需要冲突检测。

| 100GBASE-ER4 | 100 Gbps | 光纤 | 2010s |

## MAC 地址

**MAC 地址（Media Access Control Address）** 是网卡的物理地址——48 位，出厂时烧录在网卡上。

```
格式：XX:XX:XX:YY:YY:YY
      ↑厂商代码  ↑设备编号

举例：00:1A:2B:3C:4D:5E

特殊地址：
FF:FF:FF:FF:FF:FF → 广播地址（发送给所有设备）
01:00:5E:xx:xx:xx → 多播地址
```
