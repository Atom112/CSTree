## 在没有互联网的年代，两台电脑怎么通信？

把两台电脑用一根网线直接连起来——但这只是开始。它们需要在同一时间、用同一速度、用同一编码方式发送数据、确保数据不错乱、能纠正错误……

网络通信涉及的问题太多了。**分层**就是解决复杂问题的经典思路。

> 🏫 **类比：寄快递**
> 你寄一个包裹，不需要知道飞机怎么飞、卡车怎么开——你只需要写地址、贴快递单。
> 
> 网络分层也一样：应用层的数据，交给传输层封装成段，再交给网络层封装成包，再交给链路层封装成帧——每层只关心自己的事。

## OSI 七层模型

| 层 | 名称 | 功能 | 例子 |
|:-:|------|------|------|
| 7 | **应用层** | 为用户提供网络应用 | HTTP, DNS, SMTP |
| 6 | **表示层** | 数据格式转换、加密 | SSL/TLS |
| 5 | **会话层** | 建立、管理、终止会话 | NetBIOS |
| 4 | **传输层** | 端到端可靠传输 | TCP, UDP |
| 3 | **网络层** | 路由选择和寻址 | IP, ICMP |
| 2 | **数据链路层** | 帧封装、差错检测 | Ethernet, Wi-Fi |
| 1 | **物理层** | 比特流的传输 | 网线、光纤、无线电 |

> 💡 实际互联网用的是 **TCP/IP 四层模型**（应用层→传输层→网络层→网络接口层），OSI 更多是教学参考。

## TCP/IP 四层模型

```
发送方                             接收方
┌─────────┐                      ┌─────────┐
│ 应用层   │ ─→ HTTP 请求 ─→    │ 应用层   │
│ (HTTP)  │                      │ (HTTP)  │
├─────────┤                      ├─────────┤
│ 传输层   │ ─→ TCP 段 ─→      │ 传输层   │
│ (TCP)   │                      │ (TCP)   │
├─────────┤                      ├─────────┤
│ 网络层   │ ─→ IP 包 ─→       │ 网络层   │
│ (IP)    │                      │ (IP)    │
├─────────┤                      ├─────────┤
│ 网络接口 │ ─→ 以太网帧 ─→    │ 网络接口 │
│ (以太网)│                      │ (以太网)│
└─────────┘                      └─────────┘
         ↘                    ↙
          物理介质（网线/光纤）
```

### 数据封装过程

```
应用层数据：  [GET /index.html]
传输层：      [TCP头][GET /index.html]
网络层：      [IP头][TCP头][GET /index.html]
链路层：      [以太网头][IP头][TCP头][GET /index.html][校验]
```

> 💡 每一层都会加上自己的"信封"（头部）。接收方收到后，从最外层开始一层层拆封。

## 分层的好处

| 好处 | 说明 |
|------|------|
| **分工明确** | 每层只做一件事 |
| **独立演化** | 应用层不用管底层是光纤还是 5G |
| **复用** | TCP 可以为 HTTP、FTP、SMTP 等不同应用提供服务 |
| **故障隔离** | 网络问题可以定位到具体层 |

## 小结

| 概念 | 要点 |
|------|------|
| **分层模型** | 将网络通信分解为多个独立层次 |
| **OSI 七层** | 理论参考模型（物理→应用） |
| **TCP/IP 四层** | 实际互联网使用的模型 |
| **封装** | 每层添加头部，层层包裹数据 |
| **解封装** | 接收端从下往上逐层剥除头部 |
