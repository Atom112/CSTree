## 不能发得太快——两个原因

**流量控制**：接收方的缓冲区可能满了（比如手机在弱网环境）
**拥塞控制**：网络中间的路由器可能缓存不够（大量数据同时经过同一路由器）

> 🏫 **类比：两条水管**
> - 流量控制 = 水龙头远端的水桶快满了 → 关小水龙头
> - 拥塞控制 = 水管的中间段太细 → 进水口要慢点放

## 流量控制——滑动窗口

TCP 的**接收窗口（Receiver Window, rwnd）** 告诉发送方"我还有多少缓存空间"。

```
发送方                         接收方
  │                              │
  │ ←─── 窗口 = 10000 字节 ────→│ ← 初始，缓冲区全空
  │                              │
  │ ──── 数据 5000 字节 ──────→ │
  │                              │ 应用只处理了 2000 字节
  │ ←─── 窗口 = 7000 字节 ────→│
  │                              │
  │ ──── 数据 7000 字节 ──────→ │
  │                              │ 缓冲区满了！
  │ ←─── 窗口 = 0 字节 ────────│ ← 发送方停止发送！
  │                              │
  │                              │ 应用处理了 5000 字节
  │ ←─── 窗口 = 5000 字节 ────→│ ← 窗口更新，继续发
```

> 💡 窗口 = 0 时发送方停止发送——但会定期发"窗口探测"包，看接收方是否又有空间了。

## 拥塞控制——TCP 的"温柔"一面

拥塞控制比流量控制更复杂——发送方不知道网络中间发生了什么，只能通过丢包和延迟来"猜测"网络状况。

### 慢启动（Slow Start）

新连接建立后，发送方从很小的拥塞窗口（cwnd）开始，每收到一个 ACK 窗口翻倍——指数增长。

```
cwnd 变化：
1 → 2 → 4 → 8 → 16 → 32 → ...（指数增长，直到慢启动阈值）
```

### 拥塞避免

到达慢启动阈值后，每收到一个 ACK，cwnd 增加 1/cwnd——线性增长。

```
cwnd: 32 → 33 → 34 → 35 → ...（线性增长）
```

### 快速恢复

收到 3 次 DupACK → 阈值减半，cwnd 设为阈值 + 3，进入拥塞避免。

```
TCP 拥塞控制的状态机：
                ┌──────────────┐
                │   慢启动      │ ← 指数增长
                │  cwnd ×= 2   │
                └──────┬───────┘
                       │ cwnd > 阈值
                       ▼
                ┌──────────────┐
                │  拥塞避免     │ ← 线性增长
                │  cwnd += 1   │
                └──────┬───────┘
                       │ 3× DupACK
                       ▼
                ┌──────────────┐
                │  快速恢复     │ ← 减半 + 拥塞避免
                │  cwnd /= 2   │
                └──────────────┘
```

### TCP 拥塞控制的"锯齿轮"形状

```
cwnd
  ^
32│    ╱╲        ╱╲        ╱╲
```

## 拥塞控制算法对比

| 算法 | 特点 | Linux 默认 |
|------|------|:----------:|
| **Reno** | 经典算法，3 次 DupACK 触发 | ❌ |
| **CUBIC** | 适合高带宽网络（BIC 的改进） | ✅（内核 2.6.19+） |
| **BBR** | Google 开发，基于带宽和 RTT 测量 | 可选 |
| **PRR** | 快速恢复的改进 | 配合 CUBIC |

```bash
# 查看当前拥塞控制算法
$ sysctl net.ipv4.tcp_congestion_control
net.ipv4.tcp_congestion_control = cubic
```

## 小结

| 概念 | 说明 |
|------|------|
| **流量控制** | 接收窗口（rwnd）防止接收方缓冲区溢出 |
| **拥塞控制** | 拥塞窗口（cwnd）防止网络过载 |
| **慢启动** | 新连接从 1 开始指数增长 |
| **拥塞避免** | 线性增长直到丢包 |
| **快速恢复** | 丢包后减半窗口，进入拥塞避免 |
| **CUBIC** | Linux 默认的拥塞控制算法 |
