## 集线器 vs 交换机

**集线器（Hub）**：收到数据包后，复制到所有端口——不管数据是不是发给你的。
**交换机（Switch）**：记录每台电脑在哪个端口——只把数据发给目标设备。

```
Hub：                          Switch：
A 发数据给 C：                  A 发数据给 C：
  A→┌─┐                          A→┌─┐
     │H│→B （B 收到了）              │S│→C （只有 C 收到）
     │U│→C （C 收到了）              │W│  其他端口什么都没有
     │B│→D （D 收到了…）             └─┘
     └─┘
```

> 🏫 **类比：教室传纸条**
> - **集线器**：传纸条的人喊"XXX 的信！"——全班都听到了，只有 XXX 该看，但所有人都被打扰了。
> - **交换机**：传纸条的人直接走到 XXX 座位前给他——其他人完全不受影响。

## 交换机的自学习

交换机启动时"什么都不知道"——但它能通过观察数据包的源 MAC 地址"学会"每台设备的位置。

```
交换机 MAC 地址表（初始为空）：

步骤 1：A 发数据给 C（源 MAC = A, 目标 MAC = C）
  → 交换机不知道 C 在哪个端口
  → 向所有端口广播（除了 A 来的端口）
  → 记录：A 在端口 1
  ┌──────────────┐
  │ MAC    │端口  │
  │ A      │ 1    │
  └──────────────┘

步骤 2：C 回复 A（源 MAC = C, 目标 MAC = A）
  → 交换机知道 A 在端口 1 → 只发端口 1
  → 记录：C 在端口 3
  ┌──────────────┐
  │ MAC    │端口  │
  │ A      │ 1    │
  │ C      │ 3    │
  └──────────────┘

步骤 3：以后 A 发给 C → 只发端口 3
```

> 💡 交换机表项有**老化时间**（通常 300 秒）——如果一段时间没收到某 MAC 的包，就删除该表项。这样当设备移动端口时，交换机能重新学习。

## VLAN——虚拟局域网

**VLAN（Virtual LAN）** 在一台交换机上划分多个虚拟网络：

```
一台交换机，划分两个 VLAN：

VLAN 10（财务部）：端口 1, 2, 3
VLAN 20（技术部）：端口 4, 5, 6

财务的广播不会到技术那边，技术的广播也不会到财务那边。
两个 VLAN 之间不能直接通信（需要通过路由器）。
```

## 小结

| 概念 | 要点 |
|------|------|
| **集线器** | 复制数据到所有端口（低效） |
| **交换机** | 只发数据到目标端口（高效） |
| **自学习** | 从源 MAC 地址学习设备位置 |
| **MAC 地址表** | MAC→端口映射，有老化机制 |
| **VLAN** | 物理上同一交换机，逻辑上隔离的网络 |
