## IPv4 地址已经不够用了

IPv4 只有约 43 亿个地址——全球人口已经超过 80 亿。虽然 NAT 和私有地址延缓了耗尽，但 IPv6 才是根本解决方案。

> 🏫 **类比：手机号位数**
> 以前 7 位手机号就够了——但随着用户增多，升到 8 位、11 位。IPv4 → IPv6 就像手机号从 6 位升到 18 位，永远不用担心号码不够用。

## IPv6 地址

IPv6 地址 128 位——是 IPv4 的 2^96 倍。

```
IPv4：  32 位 → 4,294,967,296 个地址
IPv6： 128 位 → 340,282,366,920,938,463,463,374,607,431,768,211,456 个地址
               ≈ 地球每平方米 6.5×10^23 个地址
```

### 表示法

```
完整：2001:0db8:85a3:0000:0000:8a2e:0370:7334
简化：2001:db8:85a3::8a2e:370:7334  （去掉前导零，:: 表示连续的零）
```

### 地址类型

| 类型 | 前缀 | 说明 |
|------|:----:|------|
| **单播（Unicast）** | 2000::/3 | 全球唯一地址，类似 IPv4 公网地址 |
| **链路本地** | fe80::/10 | 类似 169.254.x.x，仅用于同一链路 |
| **唯一本地** | fc00::/7 | 类似私有地址（192.168.x.x） |
| **多播（Multicast）** | ff00::/8 | 一对多通信（替代 IPv4 广播） |

## IPv6 vs IPv4 包头对比

```
IPv4 包头：20-60 字节       IPv6 包头：40 字节（固定）
┌──────────────┐           ┌──────────────┐
│ 版本/服务类型  │           │ 版本/流量标签  │
│ 总长度/分片    │           │ 负载长度      │
│ TTL/协议/校验  │           │ 下一头部/TTL  │
│ 源地址（32位） │           │ 源地址（128位）│
│ 目标地址（32位）│           │ 目标地址（128位）│
│ 选项（可变）   │           └──────────────┘
└──────────────┘
```

**IPv6 简化的关键：**
- 没有校验和（由链路层和传输层处理）
- 没有分片（发送方在发送前就分好片）
- 没有选项（选项通过"扩展头部"实现）

## ICMP——互联网控制消息协议

**ICMP（Internet Control Message Protocol）** 用于传递网络层的控制信息和错误报告。

### 常见的 ICMP 消息

```bash
ping www.google.com
PING www.google.com (142.250.80.4): 56 data bytes
64 bytes from 142.250.80.4: icmp_seq=0 ttl=118 time=12.5 ms
64 bytes from 142.250.80.4: icmp_seq=1 ttl=118 time=11.8 ms
```

| ICMP 类型 | 用途 |
|:---------:|------|
| **0/8** | Echo Reply / Echo Request（ping） |
| **3** | 目标不可达（Dest Unreachable） |
| **5** | 重定向（Redirect） |
| **8** | Echo Request |
| **11** | 超时（TTL exceeded——traceroute 使用） |

### Traceroute

```bash
traceroute google.com
traceroute to google.com (142.250.80.4), 30 hops max
 1  192.168.1.1 (192.168.1.1)  1.5 ms
 2  10.0.0.1 (10.0.0.1)  5.2 ms
 3  172.16.1.2 (172.16.1.2)  8.1 ms
 ...
 8  142.250.80.4  12.5 ms
```

> 💡 traceroute 通过发送 TTL=1、TTL=2、TTL=3… 的包，每跳路由器都会把 TTL 减到 0 并返回 ICMP 超时消息——从而发现路径上的所有路由器。

## 小结

| 概念 | 要点 |
|------|------|
| **IPv6** | 128 位地址空间，解决地址耗尽 |
| **IPv6 简化** | 固定头部、无校验和、无路由器分片 |
| **ICMP** | 错误报告和诊断工具（ping, traceroute） |
| **IPv6 特点** | 无 NAT、自动配置、内置安全（IPSec） |
