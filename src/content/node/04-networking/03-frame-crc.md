## 为什么需要帧？

物理层只负责发送比特流——但它不区分"哪一段比特是一份完整的数据"。**帧（Frame）** 就是数据链路层的"信封"。

> 🏫 **类比：快递包裹**
> 一堆货物散装运输（比特流）→ 容易丢、容易混。每个货物打好包（帧）→ 贴上面单（头部）、写上收货地址（MAC 地址）、贴上易碎标签（校验码）。

## 以太网帧格式

```
前导码 (7B) | 定界符 (1B) | 目标 MAC (6B) | 源 MAC (6B) | 类型 (2B) | 数据 (46-1500B) | 校验 (4B)
```

- **前导码**：让接收方同步时钟
- **MAC 地址**：网卡的物理地址
- **类型**：上层协议（IPv4 = 0x0800, ARP = 0x0806）
- **数据**：IP 包（最小 46B，最大 1500B = MTU）
- **校验**：CRC-32 校验码

## 差错检测——CRC

**循环冗余校验（Cyclic Redundancy Check, CRC）** 是数据链路层使用的差错检测方法——把数据看作一个大整数，除以一个约定的多项式，余数就是校验码。

```
数据：11010011101100
生成多项式：1011

计算过程（二进制除法）：
        1100001010
1011 ) 11010011101100
       1011
       ----
        1100
        1011
        ----
         1111
         1011
         ----
          1001
          1011
          ----
           100 → 余数 = 100
```

**发送方：** 数据 + CRC 校验码一起发送
**接收方：** 重新计算 CRC，如果结果不为 0 → 数据损坏

> 💡 CRC-32 能检测出所有单比特错误、所有双比特错误、所有奇数个比特错误、以及所有长度 ≤ 32 的突发错误。

## MTU 与分片

**最大传输单元（Maximum Transmission Unit, MTU）** 是帧能承载的最大数据量（以太网是 1500 字节）。

如果 IP 包大于 1500 字节 → 必须在发送前分片：

```
IP 包 4000 字节
        │
        ▼
  ┌────────────┐
  │ 帧 1: 1500B│ ← IP 分片 1
  │ 帧 2: 1500B│ ← IP 分片 2
  │ 帧 3: 1000B│ ← IP 分片 3（最后一片）
  └────────────┘
```

## 小结

| 概念 | 要点 |
|------|------|
| **帧** | 数据链路层的数据单元 |
| **MAC 地址** | 网卡的物理标识 |
| **CRC** | 循环冗余校验，检测传输错误 |
| **MTU** | 最大传输单元（以太网 1500B） |
| **分片** | 大 IP 包拆成多个帧传输 |
