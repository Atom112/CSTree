## 为什么要划分子网？

一个 C 类网络有 254 个可用地址（192.168.1.0/24）。但有些部门只用 10 台电脑——254 个地址太浪费了。

**子网划分**就是把大网络切成更小的子网。

> 🏫 **类比：学校分班**
> 一个年级 200 人（一个 C 类网）——但你不能让 200 人都在同一间教室上课。
> 划分为 5 个班，每班 40 人（子网划分）。每个班有独立的教室（广播域），班与班之间通过走廊连接（路由器）。

## 子网掩码

**子网掩码（Subnet Mask）** 告诉计算机"IP 地址的哪部分是网络号，哪部分是主机号"：

```
IP 地址：    192.168.1.0
子网掩码：   255.255.255.0    → /24

二进制：
IP：   11000000 10101000 00000001 00000000
掩码： 11111111 11111111 11111111 00000000
       ←────── 网络号 ──────→ ← 主机号 →
```

### 划分子网示例

将 192.168.1.0/24 划分为 4 个 /26 子网：

```
子网 1：192.168.1.0/26   (192.168.1.0 - 192.168.1.63)
子网 2：192.168.1.64/26  (192.168.1.64 - 192.168.1.127)
子网 3：192.168.1.128/26 (192.168.1.128 - 192.168.1.191)
子网 4：192.168.1.192/26 (192.168.1.192 - 192.168.1.255)

每个子网有 62 个可用主机地址（64 - 2，减去网络地址和广播地址）。
```

## CIDR——无类别域间路由

**CIDR（Classless Inter-Domain Routing）** 在 1993 年提出，取代了传统的 A/B/C 类地址分类。

```
传统分类：192.168.1.0 是 C 类网络，掩码固定 255.255.255.0
CIDR：    192.168.1.0/24 或 10.0.0.0/8 或 172.16.0.0/20

关键创新：掩码可以是任意长度，不限于 8/16/24
```

**路由聚合（Route Aggregation）**——把多个小网络合成一个大网络，减少路由表：

```
多个连续网络：
  192.168.0.0/24
  192.168.1.0/24
  192.168.2.0/24
  192.168.3.0/24

聚合为一条路由：
  192.168.0.0/22（覆盖了 4 个 /24）
```

## 常见 CIDR 子网大小

| CIDR | 子网掩码 | 可用地址 | 用途 |
|:----:|:--------:|:--------:|------|
| /30 | 255.255.255.252 | 2 | 点对点链路 |
| /29 | 255.255.255.248 | 6 | 小型网络 |
| /28 | 255.255.255.240 | 14 | 小部门 |
| /24 | 255.255.255.0 | 254 | 常见局域网 |
| /16 | 255.255.0.0 | 65534 | 大型网络 |
| /8 | 255.0.0.0 | 16777214 | 超大型网 |

## 小结

| 概念 | 要点 |
|------|------|
| **子网划分** | 把大网络切分为多个小网络 |
| **子网掩码** | 区分网络号和主机号 |
| **CIDR** | 无类别的地址分配方式（IP/前缀长度） |
| **路由聚合** | 合并连续网络，减少路由表条目 |
| **VLSM** | 可变长子网掩码，不同子网不同大小 |
