## 一个公网 IP，养活全宿舍

你宿舍里：你的电脑、手机、平板、室友的电脑、室友的手机……所有设备都需要上网。但你的路由器只有一个公网 IP——怎么办？

**NAT（Network Address Translation，网络地址转换）** 就是解决这个问题的核心机制。它让所有设备"共享"同一个公网 IP 上网。

> 🏫 **类比：公司的总机电话**
> 公司只有一个总机号码（公网 IP），但每个人都有分机号（私有 IP + 端口号）：
> - 外线打进来说"转分机 123"→ 总机转接（NAT 查表找到对应的内网设备）
> - 内线打出去显示总机号码（NAT 把私有 IP 换成公网 IP）
> 
> 分机号不需要全世界知道——只要总机知道自己就行。

## NAT 的工作原理

NAT 路由器维护一张**转换表**，记录每个内网连接对应的外网端口：

```
宿舍网络拓扑：
┌───────────────────────────┐
│  路由器（公网 IP: 203.0.113.1） │
│  ┌────────────────────────┐ │
│  │ NAT 转换表              │ │
│  │ 内网:端口    │ 外网:端口  │ │
│  │─────────────│──────────│ │
│  │ 192.168.1.2:54321 │ 203.0.113.1:10001│ │
│  │ 192.168.1.3:12345 │ 203.0.113.1:10002│ │
│  │ 192.168.1.4:8888  │ 203.0.113.1:10003│ │
│  └────────────────────────┘ │
└───────────────────────────┘
         │       │       │
    你的电脑   手机    平板（各用各的分机号）
  192.168.1.2  .1.3    .1.4
```

当你手机（192.168.1.3:12345）访问百度时：
```
① 手机发数据：源 = 192.168.1.3:12345 → 目标 = 百度
② 路由器收到后，查 NAT 表：还没有 192.168.1.3:12345 的条目
③ 创建一个：192.168.1.3:12345 ↔ 203.0.113.1:10002
④ 重写 IP 包头的源地址和端口改成 203.0.113.1:10002 → 发出去
⑤ 百度回复：目标 = 203.0.113.1:10002
⑥ 路由器收到，查 NAT 表：10002 → 192.168.1.3:12345
⑦ 重写目标地址为 192.168.1.3:12345 → 转发给手机
```

**整个过程手机完全不知道 NAT 的存在**——它以为自己直接和百度通信。

### NAT 的问题

| 问题 | 说明 | 影响 |
|------|------|------|
| **无法主动访问** | 外网不能主动连接内网设备 | 你不能从学校外面直接访问宿舍的电脑 |
| **破坏端到端** | 互联网设计的"任何节点可互连"原则被破坏 | 有些 P2P 应用需要 NAT 穿透 |
| **连接数限制** | NAT 表有大小限制 | 一个路由器通常支持几百个连接 |

> 💡 **端口转发（Port Forwarding）**可以解决第一个问题：在路由器上配置"外网 8080 端口 → 内网 192.168.1.5:80"，这样外网可以通过公网 IP:8080 访问到内网服务器的 80 端口。

## DHCP——自动配置网络

如果没有 DHCP，你每次连 WiFi 都要手动输入：

```
IP 地址：    192.168.1.5
子网掩码：   255.255.255.0
默认网关：   192.168.1.1
DNS 服务器： 8.8.8.8
```

**DHCP（Dynamic Host Configuration Protocol）** 自动完成这一切——你只需要输一次 WiFi 密码。

### DORA——四步自动分配

```
你的手机                   路由器（DHCP 服务器）
  │                              │
  │ ① DISCOVER                    │
  │ "有人能给我个IP地址吗？"      │
  │ ───────────────────────────→ │
  │                              │
  │ ② OFFER                      │
  │ "用 192.168.1.5 吧？"        │
  │ ←──────────────────────────── │
  │                              │
  │ ③ REQUEST                    │
  │ "好的，我要这个IP！"          │
  │ ───────────────────────────→ │
  │                              │
  │ ④ ACK                        │
  │ "OK，分配给你了。             │
  │  网关：192.168.1.1            │
  │  DNS：8.8.8.8                │
  │  租期：24小时"               │
  │ ←──────────────────────────── │
```

### DHCP 租约

DHCP 分配的 IP **不是永久的**——而是"租赁"的，有使用期限：

```bash
# Windows 查看 DHCP 租约
ipconfig /all
# 租约获取时间：2026-06-12 10:00:00
# 租约过期时间：2026-06-15 10:00:00

# 手动释放和续约
ipconfig /release    # 归还 IP（"我不租了"）
ipconfig /renew      # 申请新 IP（"我要租一个"）
```

> 🏫 **类比：酒店入住**
> DHCP 自动分配就像酒店前台：
> - 拿出身份证（DHCP Discover）
> - "有房吗？"（DHCP Offer——"有，标间 305"）
> - "我要了"（DHCP Request）
> - "好的，这是房卡，明天 12 点前退房"（DHCP Ack + 租期）
>
> 到期了可以续租——也可以 check-out 退房（release）。

## 你是 NAT 还是公网 IP？

```bash
# 你的电脑拿到的多半是私有 IP（192.168.x.x、10.x.x.x、172.16.x.x）
ipconfig               # Windows
ip addr show           # Linux

# 你真正的公网 IP 要问路由器
# 登录路由器管理页面（通常是 192.168.1.1）查看

# 或者问百度
curl ifconfig.me       # 返回你的公网 IP
```

> 💡 如果你拿到的是 `100.x.x.x`——这是运营商级 NAT（CGNAT）。你的 ISP 把大量用户放在**一个**公网 IP 后面，意味着你不能在家里搭建外部可访问的服务器。

## 小结

| 概念 | 要点 | 一句话 |
|------|------|--------|
| **NAT** | 多个私有 IP 共享一个公网 IP | "公司总机转分机" |
| **NAT 转换表** | （私有 IP:端口）↔（公网 IP:端口） | "分机一一对应的记录" |
| **NAT 的问题** | 外网不能主动访问内网 | "只能打出去，不能打进来" |
| **DHCP** | 自动分配 IP、掩码、网关、DNS | "酒店前台登记入住" |
| **DORA** | Discover → Offer → Request → Ack | "有空房吗→有的→我要了→OK" |
| **租约** | IP 地址有使用期限，到期需续约 | "明天 12 点前退房" |

**为什么先学这个？** NAT 和 DHCP 是你每天上网都在用的技术——没有它们，连 WiFi 还需要手动配 IP。下一节进入传输层——[[udp|UDP 协议]]。
