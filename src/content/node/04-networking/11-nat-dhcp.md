## NAT——一个公网 IP 养活全家

你的手机、电脑、平板、电视都在你家——但你的路由器只有一个公网 IP。所有设备通过**NAT（Network Address Translation）** 共享这一个 IP 上网。

> 🏫 **类比：公司的总机**
> 公司有一部总机号码（公网 IP），但每个人有分机号（私有 IP + 端口）。外线打进来说明转分机号，内线打出显示总机号码。NAT 就是这部"总机"。

## NAT 的工作原理

NAT 路由器维护一张**转换表**，把（私有 IP:端口）映射为（公网 IP:端口）：

```
内部设备        转换前               转换后
手机          192.168.1.2:54321 → 203.0.113.1:10001
笔记本        192.168.1.3:12345 → 203.0.113.1:10002
平板          192.168.1.4:8888  → 203.0.113.1:10003

NAT 转换表：
┌──────────────────┬──────────────────┐
│ 内部地址:端口     │ 外部地址:端口     │
├──────────────────┼──────────────────┤
│ 192.168.1.2:54321│ 203.0.113.1:10001│
│ 192.168.1.3:12345│ 203.0.113.1:10002│
│ 192.168.1.4:8888 │ 203.0.113.1:10003│
└──────────────────┴──────────────────┘
```

**重要问题：** NAT 破坏了"端到端"原则——外部不能主动访问内部设备（除非配置端口转发）。

## DHCP——自动配置网络

没有 DHCP 时，你要手动设置 IP、子网掩码、网关、DNS……DHCP 自动完成这一切。

> 🏫 **类比：酒店入住登记**
> 你到酒店前台（DHCP 服务器），前台分配房卡（IP 地址）给你，告诉你Wi-Fi密码（DNS 服务器）、餐厅在哪（网关）。退房时归还房卡（释放 IP）。

### DHCP 工作流程（DORA）

```
设备               DHCP 服务器
 │                      │
 │ 1. DHCP Discover     │
 │    "谁在发 IP？"    → │
 │                      │
 │ 2. DHCP Offer        │
 │ ← "用 192.168.1.5?" │
 │                      │
 │ 3. DHCP Request      │
 │    "我要这个 IP！"  → │
 │                      │
 │ 4. DHCP Ack          │
 │ ← "好的，给你！"     │
 │   （含 IP、掩码、    │
 │    网关、DNS）       │
```

### DHCP 租约

IP 地址不是永久分配的——而是**租用**的：

```bash
# Windows 查看 DHCP 租约
ipconfig /all
# 租约获取时间：2026-06-12 10:00:00
# 租约过期时间：2026-06-15 10:00:00

# 更新租约
ipconfig /renew

# 释放租约
ipconfig /release
```

## 小结

| 概念 | 要点 |
|------|------|
| **NAT** | 多个私有 IP 共享一个公网 IP 上网 |
| **NAT 表** | （私有 IP:端口）↔（公网 IP:端口）映射 |
| **NAT 缺点** | 破坏端到端通信，外部不能主动访问内部 |
| **DHCP** | 自动分配 IP 地址 |
| **DORA** | Discover → Offer → Request → Ack |
| **租约** | IP 地址有使用期限，到期需续约 |
