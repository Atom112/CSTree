## 🚗 "主干道太堵了"——以太坊的扩容困境

以太坊 L1（主链）每次只能处理大约 15 笔交易/秒。当需求高时：

```
Gas 费飙升：一次简单转账可能花 $50+
确认时间变长：可能要等几小时
对于一些应用（游戏、小额支付）——完全不可用
```

**扩容（Scaling）** 就是在不牺牲去中心化和安全性的前提下，提高交易吞吐量。

> 📐 **不可能三角**：去中心化、安全、可扩展——最多同时满足两个。
>
> L1 选择了去中心化和安全——所以慢。L2 的方案是：在 L1 的安全之上，附加一个"快车道"。

## 🗂️ Rollup——当前最主流的 L2 方案

Rollup 的核心思路：**把交易"压缩打包"后发到 L1。**

```
普通交易（L1）：
用户1: A→B 1 ETH     → 直接写入 L1    → 15 TPS
用户2: B→C 0.5 ETH   → 每笔独立处理
用户3: A→D 2 ETH     → 贵 + 慢

Rollup（L2）：
用户1: A→B 1 ETH     
用户2: B→C 0.5 ETH   → 在链下打包 → 一笔提交 → 2000+ TPS
用户3: A→D 2 ETH            ↑         → 便宜 + 快
                      压缩数据 + 证明
```

### Optimistic Rollup（乐观 Rollup）

```
假设所有交易都是诚实的（乐观假设）
提交到 L1 后——有 7 天挑战期
如果在此期间有人能提供"欺诈证明"→ 交易被回滚 + 作恶者受罚

代表：Arbitrum、Optimism

优点：兼容 EVM（以太坊虚拟机）——现有合约可以直接迁移
缺点：7 天才能从 L2 提款回 L1（因为挑战期）
```

### ZK Rollup（零知识 Rollup）

```
不是"假设诚实"——用零知识证明验证
每次提交到 L1 时——附带一个"数学证明"
L1 在毫秒级验证证明 → 确认所有交易正确

代表：zkSync、StarkNet

优点：即时确认、安全性更高（不依赖"挑战期"）
缺点：和 EVM 兼容性较差，生成 zk 证明需要大量计算
```

## 🔀 其他扩容方案

| 方案 | 原理 | 优点 | 缺点 | 代表 |
|:----:|:----:|:----:|:----:|:----:|
| **状态通道** | 链下双向支付通道 | 极快、费用极低 | 功能有限 | Lightning（比特币）|
| **侧链** | 独立链 + 桥 | 开发者自由 | 安全性低于 L1 | Polygon |
| **分片** | L1 拆成多条平行链 | 扩大 L1 容量 | 实现复杂 | Ethereum 2.0（未来）|

**状态通道——以 Bitcoin Lightning Network 为例**：

```
Alice 和 Bob 打开一个支付通道 → 把资金锁定在链上
之后：Alice→Bob 转 100 次——都在链下签名即可——零确认时间、零 Gas 费
最终：关闭通道 → 把最终状态提交到链上——只花两次 Gas（开+关）
```

## 📝 小结

| 概念 | 一句话 |
|:----:|--------|
| **L2（Layer2）** | 在 L1 之上扩容——更快更便宜 |
| **Rollup** | 链下执行 + 结果提交到 L1——最主流的 L2 |
| **Optimistic Rollup** | 假设诚实 + 挑战期（7 天）— 兼容性好 |
| **ZK Rollup** | 零知识证明验证——即时确认，安全性高 |
| **状态通道** | 链下支付通道——极快，适合小额高频 |
| **L1 + L2 关系** | L1 保安全，L2 提供容量——互补 |

**为什么先学这个？** 区块链板块到此全部结束。区块链的本质是"去中心化的信任"——不依赖任何单一机构，让互不信任的参与者可以协作。未来是 L1 + L2 共同构建的 Web3 世界。
