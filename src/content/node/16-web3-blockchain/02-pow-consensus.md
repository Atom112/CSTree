## 🤔 谁来记账？——这是区块链的核心问题

去中心化的账本——没有银行这样的"中心"来决定"谁有权添加下一笔交易"。所有人都是平等的——但必须有一个方式决定"下一批交易谁记"。

**共识机制（Consensus Mechanism）** 就是解决这个问题的——让所有节点对"下一个区块由谁出"达成一致。

> 📐 **共识机制要解决的三大问题**：
> 1. 谁有权出块？（选一个记账员）
> 2. 如何防止作弊？（恶意记账怎么办？）
> 3. 如何应对分歧？（两个节点同时出块——谁的对？）

## ⛏️ PoW——工作量证明

**核心思想**："谁证明了自己做了最多工作，谁就有权出块。"

矿工要做的事：

```python
import hashlib

def mine_block(block_header, target):
    """找到一个 Nonce，使得 hash(block_header + nonce) < target"""
    nonce = 0
    while True:
        hash_result = hashlib.sha256(
            (block_header + str(nonce)).encode()
        ).hexdigest()
        
        if hash_result < target:  # 找到！出块
            return nonce, hash_result
        
        nonce += 1  # 继续尝试
        # 每秒可能尝试数万亿次
```

**难度调整**：

```
比特币每 2016 个块（约 2 周）调整一次目标值：
如果平均出块时间 < 10 分钟 → 提高难度（让 Nonce 更难找）
如果平均出块时间 > 10 分钟 → 降低难度

保持 10 分钟一个块的稳定节奏
```

**PoW 的安全保证"**：

```
攻击者想篡改区块 → 需要重新计算后续所有区块的 Nonce
→ 需要超过全网 51% 的算力
→ 成本极高 → 经济上不划算

这就是"算力即安全"——比特币的"挖矿"本质上是为网络安全付费
```

**代价**：比特币全网年耗电量 ≈ 荷兰整个国家的用电量。

## 💰 PoS——权益证明

**核心思想**：不用"算力"竞争——用"质押的金额"竞争。

```
PoW：买矿机 → 消耗电力 → 出块 → 获得奖励
PoS：质押 ETH → 被选为验证者 → 出块 → 获得奖励

PoS 的选举：
不是"所有人都能参与"——必须质押 32 ETH 成为验证者
协议随机选择下个出块的验证者——概率和质押量成正比
如果作恶——质押的 ETH 被罚没（Slashing）
```

**以太坊的 Merge（2022 年）**：

```
以太坊从 PoW 切换到 PoS 后：
能耗降低 99.9%——从"挖矿"到"质押"
出块时间从 ~13 秒降到 ~12 秒（稍快）
为后续的扩容（分片、Rollup）做准备
```

> 🏪 **类比：抽奖 vs 竞赛"
>
> **PoW** = 数学竞赛——谁解题最快谁能登台。你需要买更多书、花更多时间学习（买矿机、付电费）。
>
> **PoS** = 抽奖——谁押的钱多谁的中签率更高。但如果你作弊——押金没收。

## 📋 其他共识

| 共识 | 原理 | 适用场景 |
|:----:|:----:|:---------|
| **PoW（工作量证明）** | 算力竞争 | Bitcoin——最去中心化、最慢 |
| **PoS（权益证明）** | 质押竞争 | Ethereum 2.0——能耗低 |
| **DPoS（委托权益证明）** | 投票选出代表出块 | EOS, TRON——更快但更中心化 |
| **PBFT（拜占庭容错）** | 节点间投票 | Hyperledger Fabric——联盟链 |

## 📝 小结

| 概念 | 一句话 |
|:----:|--------|
| **PoW** | 算力竞赛——找 Nonce 使哈希小于目标值 |
| **挖矿** | 消耗算力+电力寻找有效区块 |
| **难度调整** | 保持稳定出块速度（比特币约 10 分钟）|
| **51% 攻击** | 掌握 >51% 算力可篡改链——成本极高 |
| **PoS** | 质押代币代替算力——能耗降低 99.9% |
| **Slashing** | PoS 中作恶的验证者被罚没质押 |

**为什么先学这个？** 共识决定了"谁记账"。有了共识，就可以运行[[smart-contracts|智能合约]]——在区块链上运行程序。
